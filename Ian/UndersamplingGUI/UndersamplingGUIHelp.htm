This GUI both presents and test the Undersampling fundamentals. The user sets the parameters, afterwards he can see both the spectra plot, and the list of undersamlping rates.

 

Parameters list panel

Defines the scenario under investigation.

  • Carrier frequency- the signal modulation frequency. Assumed to be much higher then
  • Channel BW- channel bandwidth. Thus the channel occupies frequencies [Carrier frequency-BW/2:Carrier frequency+BW/2] .
  • Sample frequency- a frequency chosen by the used to sample the signal with. Will be Green when no aliasing occurs and Red otherwise.
  • N-Freq. Resolution- number of points used to represent the spectra. Higher values will result in more precise graph, but will demand more resources and slow down the program.

 

Active plots list panel

Defines the spectra plots to be presented.

  • Original Spectra- presents the initial spectra we wish to sample. Fully defined by Carrier frequency and Channel BW parameters (N-Freq. Resolution defines plot resolution).
  • Resampled Spectra- wide- presents the spectra after sampling, by Sample frequency, which as you probably know causes creation of original signal copies (images) shifted by whole Sample frequency multiplications. Thus, this graph is actually a straightforward means to see if aliasing occurs, without using the undersampling condition. The red striped graph presents the limits of Nyquist BW.
  • Resampled Spectra- tight- presents the spectra after sampling inside the limits of Nyquist BW.Thus, this graph is actually a standalone version of Resampled Spectra- wide graph inside its red limits.

 

Undersampling rates table

The table presents all undersampling rates that will allow proper sampling* without aliasing.

Round table- rounds the table values to nearest integers, when enabled. While resulting in a more readable table, this however introduces inacuraties.

 

Notes.

* Note that in some cases sampling shifts signal spectra from left to right, which has to be taken care of, in order to prevent signal coruption. This will be clearly seen in Resampled Spectra- tight.

** Sometimes it is hard to see from the graphs, whether aliasing occurs. Check Sample frequency color, or increase N-Freq. Resolution.
